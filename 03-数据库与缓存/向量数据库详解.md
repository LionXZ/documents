## 1. 基础概念

### 1.1 什么是向量数据库？

向量数据库是专门用来**存储和检索向量（Embedding）**的数据库。

- 普通数据库：`SELECT * FROM products WHERE price < 100`
- 向量数据库：`找到与这张图片最相似的 10 个商品`

它的核心能力是**相似性搜索**：给定一个向量，找出数据库中与之最相似的 N 个向量。

### 1.2 核心术语

| 术语 | 说明 | 类比 |
|------|------|------|
| **Embedding / 向量** | 文本/图片/音频的数值化表示，如 [0.1, -0.3, ...] | 数字指纹 |
| **维度 (Dimension)** | 向量的长度，如 OpenAI ada-002 = 1536维 | 指纹的精度 |
| **相似度 (Similarity)** | 两个向量之间的接近程度 | 两个人的相似度评分 |
| **K-NN / Top-K** | 检索最相似的 K 个结果 | 搜出最像的前10个 |
| **ANN (近似最近邻)** | 牺牲一点点精度换巨大性能提升的检索算法 | 模糊搜索 |
| **Collection / Index** | 向量存储的基本单位，类似"表" | 表 |
| **Metadata** | 附加在向量上的结构化数据，用于过滤 | 标签 |

### 1.3 向量数据库 vs 传统数据库

| | 传统数据库 (MySQL) | 向量数据库 (ChromaDB) |
|------|-------|-------|
| **存储内容** | 结构化数据 (数字、字符串) | 向量 (浮点数组) + 元数据 |
| **查询方式** | `WHERE price < 100` | `similarity_search("好看的连衣裙")` |
| **结果** | 精确匹配 | 近似匹配（相似度排序） |
| **排序依据** | 字段值大小 | 向量距离（余弦/欧氏/点积） |
| **典型角色** | 业务数据主存储 | RAG 知识库、图片检索、推荐系统 |

### 1.4 相似度度量

```python
import numpy as np

a = np.array([0.1, 0.3, 0.5])
b = np.array([0.2, 0.3, 0.4])

# 1. 余弦相似度 (Cosine Similarity) — 最常用
#    衡量方向的一致性，值域 [-1, 1]，越大越相似
cosine_sim = np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
print(f"余弦相似度: {cosine_sim:.4f}")  # 0.9926

# 2. 欧氏距离 (Euclidean Distance) — 值越小越相似
euclidean_dist = np.linalg.norm(a - b)
print(f"欧氏距离: {euclidean_dist:.4f}")  # 0.1225

# 3. 点积 (Dot Product) — 越大越相似
dot_product = np.dot(a, b)
print(f"点积: {dot_product:.4f}")  # 0.31
```

**选型建议：**

| 场景 | 推荐度量 |
|------|---------|
| 文本检索（语义相似） | 余弦相似度 |
| 图片检索（像素值） | 欧氏距离 |
| 推荐系统（评分预测） | 点积 |

### 3.2 嵌入模型调用代码

```python
# ---- 方式1：OpenAI 嵌入 ----
from openai import OpenAI

client = OpenAI()
texts = ["什么是向量数据库？", "向量数据库如何工作？"]

response = client.embeddings.create(
    model="text-embedding-3-small",
    input=texts,
    dimensions=512,  # 可选，缩短向量维度以节省存储
)
embeddings = [d.embedding for d in response.data]
print(f"向量维度: {len(embeddings[0])}")  # 512

# ---- 方式2：本地 Sentence-Transformers (离线) ----
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("BAAI/bge-m3")  # 首次运行会自动下载
embeddings = model.encode(texts, normalize_embeddings=True)
print(f"向量维度: {len(embeddings[0])}")  # 1024

# ---- 方式3：LangChain 统一接口 ----
from langchain_openai import OpenAIEmbeddings
from langchain_huggingface import HuggingFaceEmbeddings

# OpenAI
embed_openai = OpenAIEmbeddings(model="text-embedding-3-small", dimensions=512)

# 本地模型
embed_local = HuggingFaceEmbeddings(
    model_name="BAAI/bge-m3",
    model_kwargs={"device": "cpu"},  # 或 "cuda"
    encode_kwargs={"normalize_embeddings": True},
)

# 使用
vec = embed_openai.embed_query("你好世界")
print(f"查询向量维度: {len(vec)}")

docs_vec = embed_openai.embed_documents(["文档1", "文档2", "文档3"])
print(f"文档数量: {len(docs_vec)}, 维度: {len(docs_vec[0])}")
```

### 4.3 LangChain + ChromaDB

```python
from langchain_openai import OpenAIEmbeddings
from langchain_chroma import Chroma
from langchain.text_splitter import RecursiveCharacterTextSplitter

# 1. 加载文档
with open("my_article.txt", "r") as f:
    text = f.read()

# 2. 分割
splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50,
    separators=["\n\n", "\n", "。", ".", " "],
)
chunks = splitter.create_documents([text])
print(f"分割出 {len(chunks)} 个文本块")

# 3. 创建向量库
embeddings = OpenAIEmbeddings(model="text-embedding-3-small", dimensions=512)

vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    persist_directory="./chroma_langchain",  # 持久化路径
    collection_name="my_articles",
)

# 4. 相似性搜索
docs = vectorstore.similarity_search("什么是 RAG？", k=3)
for i, doc in enumerate(docs):
    print(f"{i+1}. {doc.page_content[:100]}...")

# 5. 带分数的相似性搜索
docs_with_score = vectorstore.similarity_search_with_relevance_scores(
    "什么是 RAG？", k=3
)
for doc, score in docs_with_score:
    print(f"[相似度: {score:.3f}] {doc.page_content[:80]}...")

# 6. MMR 搜索（多样性检索 — 避免结果太雷同）
docs = vectorstore.max_marginal_relevance_search(
    "什么是 RAG？", k=3, fetch_k=10, lambda_mult=0.5
)

# 7. 元数据过滤（需要先为 chunk 添加 metadata）
docs = vectorstore.similarity_search(
    "什么是 RAG？",
    k=3,
    filter={"source": "docs/rag.md"},
)

# 8. 转为 Retriever
retriever = vectorstore.as_retriever(
    search_type="similarity",   # "similarity" / "mmr" / "similarity_score_threshold"
    search_kwargs={"k": 5},
)
docs = retriever.invoke("向量检索原理")
```

### 5.2 原生 FAISS 操作

```python
import faiss
import numpy as np

# ---- 创建索引 ----
dimension = 512                     # 向量维度
index = faiss.IndexFlatL2(dimension) # L2 距离的精确搜索索引

# 或使用内积（余弦相似度需先归一化）
# index = faiss.IndexFlatIP(dimension)

print(f"索引中的向量数: {index.ntotal}")  # 0
```

```python
# ---- 添加向量 ----
# 生成一些随机向量（实际使用时替换为你的 embedding）
np.random.seed(42)
num_vectors = 10000
vectors = np.random.random((num_vectors, dimension)).astype("float32")

# 归一化（如果使用内积做余弦相似度）
# faiss.normalize_L2(vectors)

index.add(vectors)
print(f"索引中的向量数: {index.ntotal}")  # 10000
```

```python
# ---- 搜索 ----
query = np.random.random((1, dimension)).astype("float32")
k = 5  # 返回 top-5

distances, indices = index.search(query, k)
print("查询结果:")
for i, (idx, dist) in enumerate(zip(indices[0], distances[0])):
    print(f"  {i+1}. 向量 #{idx} (距离: {dist:.4f})")
```

```python
# ---- 保存与加载 ----
# 保存索引到磁盘
faiss.write_index(index, "my_index.faiss")

# 从磁盘加载
loaded_index = faiss.read_index("my_index.faiss")
```

```python
# ---- GPU 加速 ----
# 将索引迁移到 GPU（需 faiss-gpu）
gpu_res = faiss.StandardGpuResources()
gpu_index = faiss.index_cpu_to_gpu(gpu_res, 0, index)  # 0 = GPU 编号

# 之后的搜索会自动使用 GPU
distances, indices = gpu_index.search(query, k)
```

```python
# ---- 索引工厂：一行代码创建索引 ----
import faiss

# 自动选择索引类型
dim = 512
nlist = 100

# IVF + PQ 索引（压缩存储，适合海量数据）
quantizer = faiss.IndexFlatL2(dim)
index = faiss.IndexIVFPQ(quantizer, dim, nlist, 16, 8)  # 16 子向量, 8 bits 编码
index.train(vectors)   # IVF* 需要训练
index.add(vectors)

# 或使用 index_factory 字符串描述
index = faiss.index_factory(512, "IVF100,Flat")
```

### 5.3 常用索引类型

```python
import faiss
import numpy as np
import time

d = 512
nb = 100000       # 数据库向量数
nq = 100          # 查询向量数

np.random.seed(42)
xb = np.random.random((nb, d)).astype("float32")
xq = np.random.random((nq, d)).astype("float32")

# ---- 1. IndexFlatL2 — 精确搜索，暴力遍历 ----
index = faiss.IndexFlatL2(d)
index.add(xb)
t0 = time.time()
D, I = index.search(xq, 10)
print(f"FlatL2:     {time.time()-t0:.4f}s,  内存估算: {nb*d*4/1024/1024:.0f}MB")

# ---- 2. IndexHNSWFlat — HNSW 图索引 (高召回率) ----
index = faiss.IndexHNSWFlat(d, 32)  # 32 = M, 每个节点的连接数
index.add(xb)
t0 = time.time()
D, I = index.search(xq, 10)
print(f"HNSWFlat:   {time.time()-t0:.4f}s")

# ---- 3. IndexIVFFlat — 倒排索引 + 精确搜索 ----
nlist = 100
quantizer = faiss.IndexFlatL2(d)
index = faiss.IndexIVFFlat(quantizer, d, nlist)
index.train(xb)  # 必须先训练
index.add(xb)
t0 = time.time()
index.nprobe = 10  # 搜索时探测的聚类数（默认1，太小不精确）
D, I = index.search(xq, 10)
print(f"IVF{nlist},Flat: {time.time()-t0:.4f}s")

# 索引类型选择指南:
# < 1万条     → IndexFlatL2 (够用)
# 1万-100万   → IndexHNSWFlat 或 IndexIVFFlat
# 100万-1亿   → IndexIVFPQ (压缩)
# > 1亿       → 上 Milvus (分布式)
```

### 5.4 LangChain + FAISS

```python
from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings
from langchain.text_splitter import RecursiveCharacterTextSplitter

embeddings = OpenAIEmbeddings(model="text-embedding-3-small", dimensions=512)

# ---- 创建 ----
with open("knowledge_base.txt", "r") as f:
    text = f.read()

splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.create_documents([text])

vectorstore = FAISS.from_documents(chunks, embeddings)
print(f"索引文档数: {vectorstore.index.ntotal}")

# ---- 保存 ----
vectorstore.save_local("faiss_index/")

# ---- 加载 ----
vectorstore = FAISS.load_local(
    "faiss_index/",
    embeddings,
    allow_dangerous_deserialization=True,  # ⚠️ 只加载自己创建的文件
)

# ---- 搜索 ----
docs = vectorstore.similarity_search("你的查询内容", k=5)
for doc in docs:
    print(doc.page_content[:100])

# ---- MMR 多样性搜索 ----
docs = vectorstore.max_marginal_relevance_search("你的查询内容", k=5, fetch_k=20)

# ---- 合并索引（增量更新） ----
new_chunks = splitter.create_documents(["新增的文档内容..."])
db2 = FAISS.from_documents(new_chunks, embeddings)
vectorstore.merge_from(db2)
```

## 11. RAG 完整实战

### 11.1 从零搭建 RAG 问答系统

```python
"""
RAG 完整流程:
1. 文档加载 → 2. 文本分割 → 3. 向量化 → 4. 存储
→ 5. 查询向量化 → 6. 检索 → 7. 拼接 Prompt → 8. LLM 生成
"""
import os
from dotenv import load_dotenv

load_dotenv()

# ===== Step 1: 加载文档 =====
from langchain_community.document_loaders import PyPDFLoader, TextLoader, WebBaseLoader

# PDF
loader = PyPDFLoader("knowledge_base.pdf")
# TXT
# loader = TextLoader("knowledge_base.txt")
# 网页
# loader = WebBaseLoader("https://example.com/article")

documents = loader.load()
print(f"加载了 {len(documents)} 个文档, 总长度 {sum(len(d.page_content) for d in documents)} 字符")

# ===== Step 2: 文本分割 =====
from langchain.text_splitter import RecursiveCharacterTextSplitter

text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,           # 每块最大 500 字符
    chunk_overlap=50,         # 块之间重叠 50 字符（保持上下文连贯）
    length_function=len,
    separators=["\n\n", "\n", "。", ".", " ", ""],
)
chunks = text_splitter.split_documents(documents)
print(f"分割出 {len(chunks)} 个文本块")

# ===== Step 3: 向量化 + 存储 =====
from langchain_openai import OpenAIEmbeddings
from langchain_chroma import Chroma

embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",
    dimensions=512,
)

vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    persist_directory="./rag_chroma",
    collection_name="knowledge_base",
)

# ===== Step 4: 创建 Retriever =====
retriever = vectorstore.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 4},
)

# ===== Step 5: 构建 RAG Chain =====
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_core.output_parsers import StrOutputParser

llm = ChatOpenAI(model="gpt-4o", temperature=0.3)

# Prompt 模板
template = """你是一个专业的知识问答助手。请根据以下参考信息回答用户问题。

参考信息:
{context}

用户问题: {question}

回答要求:
- 如果参考信息中有答案，请基于参考信息回答
- 如果参考信息不足，请诚实地说不知道
- 回答要简洁、准确

回答:"""

prompt = ChatPromptTemplate.from_template(template)

# 组装 Chain
def format_docs(docs):
    return "\n\n".join(f"[来源 {i+1}] {doc.page_content}" for i, doc in enumerate(docs))

rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

# ===== Step 6: 提问！ =====
questions = [
    "什么是向量数据库？",
    "RAG 的工作原理是什么？",
    "ChromaDB 有哪些使用方式？",
]

for q in questions:
    print(f"\n{'='*60}")
    print(f"Q: {q}")
    answer = rag_chain.invoke(q)
    print(f"A: {answer}")
```

## 14. 附录：向量维度和容量估算

### 14.1 存储容量计算

```python
# 向量存储容量估算
def estimate_storage(num_vectors, dimension, dtype_bytes=4):
    """
    num_vectors: 向量数量
    dimension: 维度
    dtype_bytes: float32 = 4 bytes, float16 = 2 bytes
    """
    vector_bytes = num_vectors * dimension * dtype_bytes
    metadata_bytes = num_vectors * 512  # 假设平均 512 字节 metadata

    total_gb = (vector_bytes + metadata_bytes) / (1024**3)
    return total_gb

# 例子: 100万条 text-embedding-3-small(512维), float32
gb = estimate_storage(1_000_000, 512)
print(f"100万条 512维向量 ≈ {gb:.2f} GB")

# 不同规模估算
for n in [1_000, 10_000, 100_000, 1_000_000, 10_000_000]:
    gb = estimate_storage(n, 512)
    print(f"{n:>12,} 条 ≈ {gb:>7.2f} GB")
```

### 14.2 搜索速度基准

```python
# 不同索引类型在不同规模下的性能（单线程, float32, 512维）
# 仅供参考，实际情况受硬件、查询数、并发影响

"""
| 索引类型              | 1万条     | 10万条    | 100万条   | 备注                     |
|----------------------|-----------|-----------|-----------|--------------------------|
| FAISS FlatL2         | 0.2ms     | 2ms       | 20ms      | 精确结果，可接受          |
| FAISS HNSWFlat       | 0.05ms    | 0.1ms     | 0.3ms     | 近似，召回率 ~99%        |
| FAISS IVFPQ          | -         | 0.3ms     | 1ms       | 压缩存储，召回率 ~95%    |
| Milvus HNSW          | 1ms       | 2ms       | 5ms       | 含网络开销               |
| ChromaDB (本地)       | 1ms       | 3ms       | 15ms      | 嵌入式，数据库线程开销    |
| Pinecone (p1)        | 10ms      | 15ms      | 20ms      | 含网络延迟               |
| pgvector IVFFlat     | 0.5ms     | 3ms       | 15ms      | 受 PG 负载影响           |

* 以上为单次查询 P90 延迟，单位毫秒
* FAISS 不含网络开销，为进程内搜索
* 云服务延迟取决于你和数据中心的地理距离
"""

# 实际测试脚本
import time
import faiss
import numpy as np

np.random.seed(42)
d = 512

for n in [10000, 100000, 500000]:
    # 创建索引
    index = faiss.IndexHNSWFlat(d, 32)
    data = np.random.random((n, d)).astype("float32")
    index.add(data)

    # 测试 100 次查询
    queries = np.random.random((100, d)).astype("float32")
    start = time.time()
    for q in queries:
        index.search(q.reshape(1, -1), 10)
    avg_time = (time.time() - start) / 100 * 1000

    print(f"{n:>8} 条 → HNSWFlat 平均 {avg_time:.2f}ms/查询")
```
