# 外贸获客 Agent - 技术方案

## Context

开发一款不限行业的通用型外贸获客 Agent，实现：自动发现潜在客户 → 挖掘联系人 → AI生成个性化开发信 → 自动发送 → 追踪分析 的完整自动化闭环。

## 整体架构

```
前端管理后台 (React 19 + Vite + Tailwind CSS + shadcn/ui)
        │
API 网关 (FastAPI)
        │
┌───────┼──────────┬────────────┬──────────────┐
│ 发现服务  │ 联系人服务 │ 邮件服务     │ 追踪分析服务  │
│ Discovery│ Contact  │ Email       │ Analytics    │
└───────┼──────────┴────────────┴──────────────┘
        │
┌───────┴──────────────────────────────────────────┐
│           AI Agent 编排层 (LangGraph)              │
│  发现Agent → 联系人Agent → 邮件Agent → 回复分析Agent │
└───────────────────────┬──────────────────────────┘
                        │
┌───────────────────────┴──────────────────────────┐
│  数据层: PostgreSQL + Redis + ChromaDB(向量)      │
│  任务队列: RabbitMQ                               │
│  外部工具: SerpAPI/Playwright/Hunter.io/SMTP      │
└──────────────────────────────────────────────────┘
```

## 五大核心模块

### 1. 客户发现引擎
- **数据来源**: Google搜索(SerpAPI+Playwright)、LinkedIn、B2B平台(Alibaba)、海关数据、展会名录
- **行业配置化**: 不同行业可配置不同关键词、搜索模板、HS编码、目标国家
- **LLM结构化提取**: 用LLM从搜索摘要中提取公司名/官网/行业/规模等信息
- **客户评分模型**: 行业匹配度(25%) + 网站可信度(20%) + 规模适配(15%) + 地区匹配(15%) + 数据完整度(15%) + 数据源权重(10%)

### 2. 联系人挖掘
- **三级策略**: Hunter.io API直查 → LinkedIn爬取关键决策人 → 域名推测邮箱格式+SMTP验证
- **决策人识别**: 自动区分关键决策人(CEO/Founder)、决策人(Purchasing/Sourcing Director)、相关人员(Manager)
- **邮箱验证**: SMTP握手验证 + ZeroBounce/NeverBounce API + 语法校验

### 3. AI邮件生成
- **LLM个性化**: 根据公司背景、行业痛点、收件人职位生成个性化开发信
- **模板系统**: 多种邮件框架(问题导向型/价值主张型)，存ChromaDB做向量检索
- **A/B测试**: 自动分两组发送不同模板，统计胜出方案
- **多语言支持**: 英语/西语/德语/日语/阿拉伯语等
- **反垃圾规则**: 主题<50字符、无全大写、无过度标点、真实发件人信息

### 4. 邮件发送引擎
- **多邮箱池**: 多账号负载均衡，分散发送量
- **预热策略**: 新邮箱从5封/天起步，30天逐步增加到200封/天
- **频率控制**: 每封间隔30-180秒随机延迟
- **退信处理**: 硬退信标记无效、软退信24小时后重试(最多3次)、投诉自动退订
- **推荐**: Google Workspace(150-200封/天)、SendGrid(免费100封/天)、Resend(开发者友好)

### 5. 追踪与分析
- **打开追踪**: 1x1透明像素图片
- **点击追踪**: 链接重定向中间页
- **回复分类**: LLM自动识别(感兴趣/拒绝/自动回复/询问详情/转介绍/退订)
- **客户评分更新**: 基础分(40%) + 互动分(60%)，自动计算跟进优先级

## 技术栈

| 层级 | 技术 |
|------|------|
| Agent框架 | LangChain 1.3+ + LangGraph 1.2+ |
| 后端 | Python FastAPI 0.115+ + SQLAlchemy 2.0 async |
| 数据库 | PostgreSQL 16 + Redis 7 + ChromaDB |
| 任务队列 | Celery + RabbitMQ |
| 前端 | Vite 6 + React 19 + Tailwind CSS 4 + shadcn/ui |
| LLM | GPT-4o (主力) + GPT-4o-mini (分类/摘要) |
| 浏览器自动化 | Playwright (Python) |
| 部署 | Docker + docker-compose |

## 数据库核心表

- `companies` - 客户公司(名称/官网/国家/行业/AI评分/来源)
- `contacts` - 联系人(姓名/职位/邮箱/决策层级/LinkedIn)
- `campaigns` - 营销活动(行业/目标国家/模板/日限额/AB测试配置)
- `campaign_contacts` - 发送队列(发送状态/追踪/回复分类)
- `email_templates` - 邮件模板(主题/正文/行业/语种/变量)
- `sender_accounts` - 邮箱池(状态/日限额/信誉分)
- `industry_configs` - 行业配置(关键词/HS编码/决策人职位)
- `engagement_logs` - 互动记录时间线

## 实施路线图

### 阶段1：MVP核心链路 (4周)
- W1: 项目脚手架、数据库建表、FastAPI骨架

### 阶段3：智能化扩展 (4周)
- W9: 多数据源接入(海关数据/B2B平台)
- W10: 多语言邮件、多时区策略
- W11-12: LangGraph总控编排Agent、自动跟进、多用户/权限

### 阶段4：生产就绪 (2周)
- W13-14: 生产部署、CI/CD、性能优化、安全审计

## 关键提醒

- **SPF/DKIM/DMARC** 必须配置，否则进垃圾箱
- **合规性**: 遵循CAN-SPAM(美国)、GDPR(欧盟)，提供退订链接
- **绝不使用免费邮箱**(Gmail/Yahoo)批量发冷邮件
- **预热必须做**: 新域新邮箱前4-6周逐步增加发送量

## 验证方式

- 阶段1完成后: 命令行运行完整链路(搜索"LED照明 distributor US" → 找到公司 → 找联系人 → 生成邮件 → 发送 → 追踪打开/回复)，验证闭环
- 阶段2完成后: 登录管理后台创建营销活动，查看发送进度和效果数据
- 阶段3完成后: 切换不同行业配置，验证多行业适用性
