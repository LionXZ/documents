## 1. 产品定义

一个全自动内容生产流水线：

```
AI 写小说 → AI 生成漫画分镜 → AI 配音 + 剪辑 → 自动发布到抖音/TikTok
```

目标：零人工干预，持续产出短剧/漫剧内容。

---

## 2. 核心流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 1. 选题   │ → │ 2. 写稿   │ → │ 3. 分镜   │ → │ 4. 绘图   │
│ 热点分析  │    │ 章节生成  │    │ 画面描述  │    │ AI 生图   │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                    ↓
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 7. 发布   │ ← │ 6. 包装   │ ← │ 5. 合成   │ ← │ 4. 配音   │
│ 多平台API │    │ 标题+封面 │    │ 视频剪辑  │    │ TTS 语音  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

| 步骤 | 说明 | AI 参与 |
|------|------|------|
| 选题 | 抓热点 → AI 选题建议 | LLM 分析趋势 |
| 写稿 | 大纲 → 章节 → 对话 | LLM 长篇写作 |
| 分镜 | 文字转画面描述 | LLM 场景拆解 |
| 绘图 | 角色+场景生成 | Stable Diffusion / Midjourney |
| 配音 | 角色对话转语音 | TTS (Edge-TTS / GPT-SoVITS) |
| 合成 | 图+音+字幕合成视频 | FFmpeg 自动化 |
| 包装 | AI 生成标题+封面+标签 | LLM 营销文案 |
| 发布 | 调平台 API 上传 | 脚本自动化 |

## 3. 技术选型

| 层 | 方案 | 说明 |
|------|------|------|
| **大模型** | DeepSeek / GPT-4o | 写作+分镜+标题 |
| **生图模型** | ComfyUI + SDXL / Flux | 本地部署，批量生图 |
| **角色一致性** | IP-Adapter + LoRA | 同一角色多图一致 |
| **TTS 语音** | Edge-TTS (免费) / GPT-SoVITS | 多角色音色 |
| **视频合成** | FFmpeg + Python | 图+音+字幕合成 |
| **字幕生成** | Whisper / 直接时间轴 | 对话→SRT 字幕 |
| **发布** | 各平台开放 API | 抖音/TikTok/YouTube |
| **后端** | FastAPI + Celery | 异步任务队列 |
| **数据库** | MySQL + Redis | 内容管理+缓存 |
| **前端** | Vue 3 | 管理后台 |

## 5. 发布平台接入

| 平台 | API 方式 | 难度 | 限制 |
|------|------|------|------|
| **抖音** | 开放平台 Open API | ★★★ | 需企业认证，视频 15 分钟内 |
| **TikTok** | TikTok API (Content Posting) | ★★★ | 需审核，海外账号 |
| **YouTube** | YouTube Data API v3 | ★★☆ | 每日配额 10000 单位 |
| **快手** | 开放平台 | ★★★ | 需企业认证 |
| **B站** | 开放 API | ★★☆ | 较宽松 |
| **小红书** | 无公开 API | ★★★★ | 需模拟 |

**抖音接入关键步骤：**
```
1. 注册开放平台 → 创建应用 → 获取 client_key/client_secret
2. OAuth 2.0 授权 → 获取 access_token (有效期 15 天, refresh_token 续期)
3. 调用 /video/upload/ → 先上传视频 → 再 /video/create/ 发布
```

## 7. 开发阶段

| 阶段 | 内容 | 产出 |
|------|------|------|
| **Phase 1: 写稿** | LLM 长篇写作 + 人物管理 + 情节大纲 | 能自动写完整短篇小说 |
| **Phase 2: 绘图** | ComfyUI 批量生图 + 角色 LoRA | 小说→漫画图集 |
| **Phase 3: 合成** | TTS 配音 + FFmpeg 合成 | 图+音→漫剧视频 |
| **Phase 4: 发布** | 抖音/TikTok API 对接 | 一键发布 |
| **Phase 5: 全自动** | 选题→写稿→绘图→合成→发布全流水线 | 定时自动产出 |

### Phase 1 最小可用版本

```python
# 第一周就能跑的版本：
# 1. 输入: 一句话选题
# 2. LLM 写 3 章短篇小说
# 3. 输出: Markdown 文件
```

之后逐步加上绘图、配音、合成、发布。

### 与现有 DevAssistant 的关系

可以复用 DevAssistant 的技术栈（FastAPI + LangChain + Vue 3 + MySQL），新增写稿/绘图/发布 Agent 模块，加上 Celery 任务队列即可。

### 8.1 大模型 (写稿+分镜+标题)

| 方案 | 接入方式 | 成本 | 说明 |

| **DeepSeek API** | OpenAI 兼容 HTTP | ¥2/百万 token | 写 1 章小说 (2000字) ≈ ¥0.01 |

**按日产 1 部漫剧估算 (3 章 + 分镜)：** 月均 ¥10-50。

### 8.2 图像生成

| 方案 | 接入方式 | 单图成本 | 硬件要求 |
|------|------|------|------|
| **ComfyUI + SDXL (本地)** | HTTP API (localhost:8188) | ¥0 (电费) | **RTX 4090 或更高** |
| **ComfyUI + Flux (本地)** | 同上 | ¥0 | RTX 4090 24GB+ |
| **Midjourney API** | 第三方中转 | ¥0.5-2/张 | 无 |

### 8.3 TTS 语音

## 11. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| **角色一致性差** | 同一角色每张图长得不一样 | 训练角色 LoRA (20 张图 30 分钟) |
| **平台 API 变动** | 发布失败 | 监控 + 降级为手动发布 |
| **GPU 成本高** | 无法规模化 | Phase 1 先用 API 生图，验证后再上 GPU |
| **内容审核不通过** | 视频被拒 | LLM 内容审核预处理 |
| **抖音企业认证** | 需营业执照 | 先做 B站/YouTube (个人可发布) |
