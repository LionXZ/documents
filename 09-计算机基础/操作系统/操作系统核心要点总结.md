## 一、操作系统概述

### 1.1 操作系统的定义

操作系统是控制和管理整个计算机系统硬件和软件资源、合理组织调度计算机工作与资源分配、提供给用户和其他软件方便接口和环境的**最基本的系统软件**。

**一句话**：操作系统是架在用户和底层硬件之间的桥梁。

### 1.2 操作系统的功能

| 功能 | 说明 |
|------|------|
| **处理器管理** | 分配和控制处理器（CPU调度） |
| **存储器管理** | 负责内存的分配与回收 |
| **I/O设备管理** | 负责I/O设备的分配与操纵 |
| **文件管理** | 负责文件的存取、共享和保护 |

### 1.3 操作系统的四大特征

| 特征 | 说明 |
|------|------|
| **并发** | 两个或多个事件在同一时间间隔内发生，宏观同时、微观交替 |
| **共享** | 资源共享，分为互斥共享方式（如打印机）和同时共享方式（如磁盘） |
| **虚拟** | 物理实体变为逻辑对应物。时分复用（如单核CPU运行多程序）、空分复用（如C/D/E盘分区） |
| **异步** | 进程执行不是一贯到底，走走停停，以不可预知速度向前推进 |

> **最基本特征**：并发和共享，两者互为存在条件。

### 1.4 操作系统的分类

| 类型 | 特点 | 代表 |
|------|------|------|
| **手工操作** | 人机矛盾，CPU利用率极低 | 早期 |
| **单道批处理** | 引入脱机I/O，内存仅一道程序 | - |
| **多道批处理** | 多道程序并发，OS正式诞生 | - |
| **分时操作系统** | 时间片轮转，用户交互独立 | Unix、Linux |
| **实时操作系统** | 在严格时限内处理事件 | 硬实时：导弹；软实时：12306 |

### 1.5 操作系统结构

- **传统结构**：无结构 → 模块化 → 分层式
- **客户/服务器模式**：客户机→服务器→网络系统
- **微内核结构**：将OS分为微内核和多个服务器，提高正确性/灵活性/可扩展性

### 1.6 核心概念扫盲

| 概念 | 说明 |
|------|------|
| **特权指令** | 只有内核态能执行的指令（如清内存、置时钟、修改页表） |
| **非特权指令** | 普通程序可执行的指令 |
| **系统调用** | 操作系统提供给应用程序的接口，如 `read`、`write`、`fork` |
| **原语** | 由若干指令组成，执行过程连续且不可被中断 |
| **内核态（核心态）** | CPU可访问所有数据和外围设备，权限高（ring 0） |
| **用户态** | 受限访问内存，不允许访问外围设备，权限低（ring 3） |

**用户态切换到内核态的方式**：系统调用、中断、异常

## 二、进程管理

### 2.1 进程与线程

| 维度 | 进程 | 线程 |
|------|------|------|
| 定义 | 程序在数据集上的执行过程 | 进程内的一个执行单元 |
| 资源 | **资源分配的基本单位** | 共享进程资源 |
| 调度 | 传统调度单位 | **CPU调度的基本单位** |
| 开销 | 创建/切换开销大 | 创建/切换开销小 |
| 通信 | 需IPC机制 | 直接共享内存 |

**PCB（进程控制块）关键字段**：name、status、next指针、start_addr、priority、cpu_status(堆栈)、comm_info、own_resource

### 2.2 进程状态与转换

```
新建(New) → 就绪(Ready) ⇄ 运行(Running) → 终止(Terminated)
                ↑              ↓
                └── 阻塞(Blocked) ←┘
```

### 2.3 处理机调度

#### 三个层次

| 层次 | 说明 |
|------|------|
| **高级调度（作业调度）** | 从后备队列选作业调入内存，创建进程 |
| **中级调度（内存调度）** | 从挂起队列选进程调回内存 |
| **低级调度（进程调度）** | 从就绪队列选进程分配CPU |

#### 调度方式
- **非抢占式**：只允许进程主动放弃CPU
- **抢占式**：更紧急任务到达时立即暂停当前进程

#### 调度准则

- **吞吐量** = 完成作业数 / 总时间
- **CPU利用率** = CPU有效时间 / (有效时间 + 空闲时间)
- **周转时间** = 完成时间 - 提交时间
- **带权周转时间** = 周转时间 / 实际运行时间（≥1，越小越好）
- **响应时间** = 提交到首次响应的时间

#### 调度算法对比

| 算法 | 原理 | 是否可抢占 | 特点 |
|------|------|-----------|------|
| **FCFS** | 先来先服务 | 否 | 简单、效率低 |
| **SJF** | 优先最短作业 | 不可抢占 | 平均等待/周转时间最少，但会导致"饥饿" |
| **高响应比优先** | 响应比=(等待时间+运行时间)/运行时间 | 否 | 综合考量，不会饥饿 |
| **优先级调度** | 优先级高的先执行 | 均可 | 低优先级可能饥饿 |
| **时间片轮转(RR)** | 每进程轮流执行固定时间片 | 绝对可抢占 | 公平、响应快，但频繁切换有开销 |
| **多级反馈队列** | 多优先级队列+动态调整 | 抢占式 | 兼顾长短作业，响应好，会导致饥饿 |

### 2.4 进程同步与互斥

#### 基本概念

| 概念 | 说明 |
|------|------|
| **临界资源** | 一次仅允许一个进程使用的共享资源 |
| **临界区** | 访问共享资源的代码片段 |
| **同步** | 多个进程因合作产生制约关系，有先后执行顺序 |
| **互斥** | 同一时刻只有一个进程能进入临界区 |

#### 临界区四原则

1. **空闲让进**：无进程在临界区时，允许进程进入
2. **忙则等待**：临界区忙时，其他进程必须在外面等待
3. **有限等待**：进入临界区的请求在有限时间内得到满足
4. **让权等待**：等待进程放弃CPU

#### 信号量（PV操作）

- **P操作** (wait)：S减1；若 S≥0 继续执行；若 S<0 阻塞并放入等待队列
- **V操作** (signal)：S加1；若 S>0 继续执行；若 S≤0 释放等待队列中的一个进程
- **互斥量（mutex）**：信号量取值仅为 0 或 1 的特殊情况

#### 经典问题：生产者-消费者

```c
semaphore mutex = 1;   // 临界区互斥信号量
semaphore empty = n;   // 空闲缓冲区数
semaphore full = 0;    // 满缓冲区数

// 生产者
producer() {
    while(1) {
        生产数据;
        P(empty);   // 申请空缓冲区
        P(mutex);   // 进入临界区
        放入缓冲区;
        V(mutex);   // 退出临界区
        V(full);    // 满缓冲区+1
    }
}

// 消费者
consumer() {
    while(1) {
        P(full);    // 申请满缓冲区
        P(mutex);   // 进入临界区
        取出数据;
        V(mutex);   // 退出临界区
        V(empty);   // 空缓冲区+1
        消费数据;
    }
}
```

#### 管程（Monitor）

由三部分组成：共享变量声明 + 操作过程 + 初始化语句。Java 中 `synchronized` 就是管程机制。

### 2.5 死锁

#### 产生死锁的四个必要条件

1. **互斥条件**：资源只能独占
2. **不剥夺条件**：资源未用完不能被强行夺走
3. **请求和保持条件**：持有资源的同时又申请新资源被阻塞，但不释放已有资源
4. **循环等待条件**：存在进程-资源循环等待链

#### 死锁处理策略

| 策略 | 方法 |
|------|------|
| **预防死锁** | 破坏四个必要条件之一（破坏互斥不现实，通常破坏请求保持或循环等待） |
| **避免死锁** | **银行家算法**：分配前预判是否进入不安全状态 |
| **死锁检测** | 通过资源分配图化简检测 |
| **死锁解除** | ①资源剥夺法 ②撤销进程法 ③进程回退法 |

#### 银行家算法步骤

1. 如果 `Request[i] ≤ Need[i]` → 继续，否则出错
2. 如果 `Request[i] ≤ Available` → 继续，否则等待
3. 试探分配：`Available -= Request`，`Allocation += Request`，`Need -= Request`
4. 执行安全性算法，若安全才正式分配

## 三、内存管理

### 3.1 内存管理的功能

| 功能 | 说明 |
|------|------|
| **内存分配与回收** | 合理划分和动态分配内存空间 |
| **地址转换** | 逻辑地址 → 物理地址 |
| **内存扩充** | 虚拟存储技术逻辑上扩充主存 |
| **存储保护** | 保证各作业在各自存储空间互不干扰 |

### 3.2 程序执行流程

```
源代码 → [编译] → 目标模块 → [链接] → 装入模块 → [装入] → 内存中可执行
```

**链接方式**：静态链接、装入时动态链接、运行时动态链接

**装入方式**：绝对装入、静态可重定位装入、动态运行时装入

### 3.3 覆盖与交换

| 技术 | 核心思想 | 特点 |
|------|----------|------|
| **覆盖** | 程序分段，常用段常驻内存，不常用段按需调入 | 打破进程全部装入才能运行的限制 |
| **交换** | 内存紧张时将进程换出到外存，换入已就绪进程 | 提升多道程序并发度 |

### 3.4 连续分配管理

| 方式 | 特点 | 碎片问题 |
|------|------|----------|
| **单一连续分配** | 内存分系统区和用户区，仅一道用户程序 | 有内部碎片，无外部碎片 |
| **固定分区分配** | 内存分为固定大小分区 | 有内部碎片，无外部碎片 |
| **动态分区分配** | 按进程需要动态建立分区 | 无内部碎片，有外部碎片 |

#### 动态分区分配算法

| 算法 | 原理 | 优缺点 |
|------|------|--------|
| **首次适应 (First Fit)** | 从低地址开始找第一个满足的空闲分区 | 综合性能最好，开销小 |
| **最佳适应 (Best Fit)** | 选最接近需求大小的空闲分区 | 保留大分区，但产生很多小碎片 |
| **最坏适应 (Worst Fit)** | 优先用最大的空闲分区 | 减少小碎片，但大进程分配困难 |
| **邻近适应 (Next Fit)** | 从上次结束位置开始循环查找 | 避免每次都从低地址检索 |

### 3.5 非连续分配管理

#### 分页存储管理

将内存分为大小相等的**页框**（通常 4KB），进程地址空间分为同样大小的**页**。页可不连续存放。

**地址转换**：逻辑地址 → 页号 + 页内偏移 → 查页表 → 内存块号 + 偏移 = 物理地址

**例题**：页面大小 1KB，页号 2 对应内存块号 8，逻辑地址 A=2500
- 页号 = 2500/1024 = 2，页内偏移 = 2500%1024 = 452
- 物理地址 = 8×1024 + 452 = **8644**

#### 分段存储管理

按程序逻辑关系划分为段（代码段、数据段、栈段等），每段从 0 开始编址。

#### 段页式管理

先分段，段内再分页，兼顾逻辑性和物理性。需要段表和页表两级映射。

### 3.6 虚拟内存

#### 局部性原理

| 类型 | 原理 | 例子 |
|------|------|------|
| **时间局部性** | 刚执行的指令/数据可能很快再次使用 | 循环 |
| **空间局部性** | 访问某单元的附近单元可能很快被访问 | 数组连续存储 |

基于局部性原理，无需将程序全部装入内存即可运行，用完不上的留在外存。用户看来有一个比实际内存大得多的内存 → **虚拟内存**。

#### 虚拟内存实现方式

1. 请求分页存储管理
2. 请求分段存储管理
3. 请求段页式存储管理

#### 快表（TLB）

存放在高速缓冲存储器中的部分页表，作为页表的 Cache。用快表命中时只需访问 1 次 Cache + 1 次主存（而非 2 次主存）。

#### 地址转换流程

1. 按页号查快表
2. 若命中 → 页架号 + 单元号 = 绝对地址
3. 若未命中 → 查主存页表 → 形成绝对地址 → 登记到快表
4. 快表满时按替换策略淘汰旧项

#### 页面置换算法

| 算法 | 原理 | 特点 |
|------|------|------|
| **OPT（最佳置换）** | 淘汰未来最长时间不用的页面 | 理论最优但不可实现 |
| **FIFO（先进先出）** | 淘汰最早进入内存的页面 | 简单但可能有 Belady 异常 |
| **LRU（最近最久未使用）** | 淘汰最长时间未访问的页面 | 效果好但实现开销大 |
| **LFU（最少使用）** | 淘汰近期访问次数最少的页面 | 对周期性访问不敏感 |

#### 页面置换策略

| 策略 | 说明 |
|------|------|
| **固定分配局部置换** | 分配固定物理块，缺页时只能换自己页面 |
| **可变分配全局置换** | 发生缺页就分配新物理块，但可能影响其他进程 |
| **可变分配局部置换** | 根据缺页率动态增减物理块（最优） |

#### 页面抖动

刚换出的页面马上又要调入，刚调入的马上又要换出。**原因**：分配的物理块太小，或驻留进程太多。

## 四、文件管理

### 4.1 文件与文件系统

**文件**：存储在介质上的、具有文件名的一组有序信息集合。

**文件系统**：操作系统中负责操纵和管理文件的一整套设施，实现文件的共享、保护和"按名存取"。

### 4.2 文件逻辑结构

| 结构 | 说明 | 例子 |
|------|------|------|
| **无结构（流式文件）** | 按字节流组织 | 文本文档 |
| **有结构（记录式文件）** | 由记录组成 | 数据库表 |

### 4.3 文件物理结构

| 分配方式 | 原理 | 特点 |
|----------|------|------|
| **连续分配** | 存储到连续的物理盘块 | 访问快，支持随机访问；有外部碎片，扩展困难 |
| **链接分配** | 离散放置，用链接指针连接 | 无外部碎片；随机访问慢，指针占空间 |
| **索引分配** | 建立索引表记录各块位置 | 支持随机访问，无碎片；索引块有额外开销 |

### 4.4 文件控制块（FCB）

FCB 是"按名存取"的数据结构，包含：

| 类别 | 内容 |
|------|------|
| **基本信息** | 文件名、物理地址 |
| **存取控制信息** | 文件存取权限（文件主/同组/其他用户 → rwx） |
| **使用信息** | 建立日期、修改日期、访问日期、打开进程数 |

### 4.5 目录结构

| 结构 | 特点 |
|------|------|
| **单级目录** | 简单，但不允许重名、查找慢，仅适用于单用户 |
| **多级目录（树形目录）** | 像倒置的树，从根到叶。MS-DOS、Unix 均采用 |

### 4.6 文件操作常用系统调用

| 系统调用 | 功能 |
|----------|------|
| `create` | 建立文件，检查同目录下是否有同名文件 |
| `open` | 找到 FCB 并拷贝到内存 |
| `read` / `write` | 通过 PCB 中的打开文件表指针找到盘块进行读写 |
| `close` | 将修改后的 FCB 写回磁盘，释放内存表 |
| `delete` | 删除目录项和索引节点内容 |

### 4.7 文件存储空间管理

| 方法 | 说明 |
|------|------|
| **空闲表法** | 为外存所有空闲区建立空闲表（连续分配） |
| **位示图法** | 每个盘块对应一个二进制位（0空闲/1占用） |
| **空闲块链接法** | 将空闲盘区链接成队列 |

### 4.8 磁盘调度算法

磁盘由一个磁头根据系统想要访问的内容移动到相应位置。移动顺序就是磁盘调度要解决的问题。

| 算法 | 原理 | 特点 |
|------|------|------|
| **FCFS** | 按访问请求的先后顺序 | 简单但寻道时间长 |
| **SSTF（最短寻道时间优先）** | 优先处理离当前磁头最近的磁道 | 可能产生饥饿（小区域内移动） |
| **SCAN（电梯算法）** | 磁头一直移动到最外/内侧才调头 | 防止饥饿，公平性好 |
| **LOOK** | 移动方向上无请求时立即调头 | 优化版SCAN |

## 五、I/O 管理

### 5.1 I/O 控制方式演进

| 方式 | CPU参与度 | 数据传送单位 | 说明 |
|------|-----------|-------------|------|
| **程序查询方式** | 全程参与轮询 | 字 | CPU不断查询状态寄存器，效率极低 |
| **中断方式** | 仅开始和结束参与 | 字 | I/O完成后控制器发中断信号，CPU再处理 |
| **DMA** | 仅开始和结束参与 | 数据块 | DMA控制器直接与内存交互传送整个数据块 |
| **通道方式** | 几乎不参与 | 一组数据块 | 专用I/O处理器独立执行通道程序 |

### 5.2 I/O 软件层次结构

```
用户层 I/O 软件（如 printf、scanf）
        ↓
设备独立性软件（统一接口、缓冲管理）
        ↓
设备驱动程序（与具体硬件交互）
        ↓
中断处理程序（响应硬件中断）
        ↓
硬件层（设备控制器、物理设备）
```

### 5.3 DMA 控制器关键寄存器

| 寄存器 | 作用 |
|--------|------|
| **DR（数据寄存器）** | 暂存设备与内存间的数据 |
| **MAR（内存地址寄存器）** | 标记数据在内存中的位置 |
| **DC（数据计数器）** | 剩余要读/写的字节数 |
| **CR（命令/状态寄存器）** | 存放CPU的I/O命令或设备状态 |

### 5.4 SPOOLing 技术（假脱机）

将一台物理设备逻辑上虚拟成多台设备，把**独占设备改造成共享设备**。

- **脱机技术**：提前将数据输入到磁带，主机从快速的辅存读数据
- **假脱机技术**：用软件方式模拟脱机技术，以磁盘 I/O 的高速度替代低速外设

> **关键价值**：多个进程同时使用一台独占设备时，每个进程都认为自己是独占该设备。

## 六、考试/面试重点速记

### 6.1 常考选择题/填空题

| 考点 | 答案要点 |
|------|----------|
| 时间片轮转调度 | **绝对可抢占** |
| 平均周转时间最短 | **短作业优先（SJF）** |
| 死锁必要条件 | 互斥、不剥夺、请求保持、循环等待 |
| 死锁解除方式 | 资源剥夺法、撤销进程法、进程回退法 |
| 页面抖动 | 不会导致系统崩溃（会忙碌但不崩溃） |
| 实现虚拟设备的技术 | SPOOLing |

### 6.2 常用公式

- **平均周转时间** = 各作业周转时间之和 / 作业数
- **带权周转时间** = 周转时间 / 实际运行时间
- **响应比** = (等待时间 + 运行时间) / 运行时间
- **物理地址**（分页）= 块号 × 页面大小 + 页内偏移

### 6.3 时间线：操作系统发展

```
手工操作 → 单道批处理 → 多道批处理（OS诞生）→ 分时OS → 实时OS → 现代OS
```
