## 第一章：计算机系统概述

### 1.1 冯·诺依曼结构

**五大部件**：运算器、控制器、存储器、输入设备、输出设备

| 部件 | 功能 |
|------|------|
| **运算器** | 算术/逻辑运算，核心是 ALU |
| **控制器** | 取指令、译码、自动执行 |
| **存储器** | 存放数据和指令（主存 + 辅存） |
| **输入设备** | 将外界信息输入计算机 |
| **输出设备** | 将处理结果输出 |

> 运算器 + 控制器 = **CPU**

**四大核心思想**：

1. 采用**二进制**表示指令和数据
2. **"存储程序"工作方式**——程序预先存入主存，启动后自动逐条执行
3. 硬件由五大基本部件组成
4. 指令由**操作码 + 地址码**组成

### 1.2 关键寄存器

| 寄存器 | 缩写 | 功能 |
|--------|------|------|
| 程序计数器 | **PC** | 自动存放下一条指令地址 |
| 指令寄存器 | **IR** | 临时存放从主存取来的指令 |
| 存储器地址寄存器 | **MAR** | 存放主存地址 |
| 存储器数据寄存器 | **MDR** | 存放读/写的数据 |
| 算术逻辑部件 | **ALU** | 执行算术和逻辑运算 |
| 通用寄存器组 | **GPRs** | 临时存放数据和运算结果 |

### 1.3 指令执行过程

```
取指令(IF) → 译码(ID) → PC增量 → 取操作数并执行(EX) → 写回结果(WB)
```

### 1.4 计算机系统层次结构

```
应用程序（最顶层）
    ↓
语言处理系统（编译器等）
    ↓
操作系统
    ↓
指令集体系结构 ISA（硬件与软件的分界线）
    ↓
计算机硬件（最底层）
```

### 1.5 程序设计语言

| 类型 | 特点 | 翻译工具 |
|------|------|----------|
| **机器语言** | 0/1 二进制，计算机直接执行 | 无需翻译 |
| **汇编语言** | 用助记符表示，面向机器 | 汇编程序 |
| **高级语言** | 接近自然语言（C/Java等） | 编译程序 或 解释程序 |

### 1.6 性能评价指标

| 指标 | 含义 |
|------|------|
| **字长** | 一次并行处理的数据位数 |
| **主频/时钟频率** | CPU 主脉冲信号频率（Hz），时钟周期的倒数 |
| **CPI** | 执行一条指令平均所需时钟周期数（与时钟频率无关） |
| **MIPS** | 每秒百万条指令数 |
| **MFLOPS/GFLOPS** | 浮点运算速度 |
| **响应时间** | 从作业提交到完成的时间 |
| **吞吐率** | 单位时间内完成的工作量 |

### 1.7 核心概念

| 概念 | 解释 |
|------|------|
| **ISA** | 指令集体系结构，软件与硬件的规约/界面 |
| **透明** | 某事物实际存在，但从某个角度看似乎不存在 |
| **存储程序** | 程序和数据以二进制形式预先存入主存，自动执行 |

## 第二章：数据的表示和运算

### 2.1 进位计数制

| 进制 | 基数 | 符号 |
|------|------|------|
| 二进制 | 2 | 0, 1 |
| 八进制 | 8 | 0~7 |
| 十进制 | 10 | 0~9 |
| 十六进制 | 16 | 0~9, A~F |

**互转法则**：二↔八（3位一组）、二↔十六（4位一组）、十↔任意（除基取余/乘基取整）

### 2.2 机器码表示（原码/反码/补码/移码）

| 编码 | 正数 | 负数 | 特点 |
|------|------|------|------|
| **原码** | 符号位0 + 数值位 | 符号位1 + 数值位 | 0有+0和-0两种 |
| **反码** | 同原码 | 符号位不变，数值位取反 | 0也有两种 |
| **补码** | 同原码 | 反码 + 1 | **0唯一编码，符号位参与运算** |
| **移码** | 补码符号位取反 | 补码符号位取反 | 用于浮点数阶码 |

**补码加减法**：
- `[X+Y]补 = [X]补 + [Y]补`
- `[X-Y]补 = [X]补 + [-Y]补`（[-Y]补 = [Y]补连同符号位取反再加1）
- 符号位参与运算，超过模的进位丢掉

**8位补码范围**：-128 ~ +127（比原码多表示一个 -128）

### 2.3 溢出判断

| 方法 | 溢出条件 | 示例 |
|------|----------|------|
| **单符号位法** | 符号位进位 ≠ 最高数值位进位 | 进位不同则溢出 |
| **双符号位法** | 01 = **正溢出**，10 = **负溢出** | 两个符号位异则为溢出 |

### 2.4 IEEE 754 浮点数

**通用格式**：`N = (-1)^S × M × R^E`

| 参数 | 含义 |
|------|------|
| S（数符） | 0正1负 |
| M（尾数） | 定点小数，决定精度 |
| E（阶码） | 定点整数，决定范围 |
| R（基数） | 固定为 2 |

**IEEE 754 标准格式**：

| 类型 | 数符 | 阶码 | 尾数 | 总位 | 偏置值 |
|------|------|------|------|------|--------|
| **单精度 (float)** | 1 | 8 | 23 | 32 | 127 |
| **双精度 (double)** | 1 | 11 | 52 | 64 | 1023 |

**真值公式**：
```
单精度 = (-1)^S × 1.M × 2^(E - 127)
双精度 = (-1)^S × 1.M × 2^(E - 1023)
```

> 注意：尾数隐含最高位 1（即实际精度为24位/53位），阶码用移码表示

**IEEE 754 特殊值编码（高频考点）**：

| 阶码 E | 尾数 M | 表示值 |
|--------|--------|--------|
| E=0, M=0 | 全0 | **±0** |
| E=0, M≠0 | 非全0 | 非规格化数：±0.M × 2^(1-偏置) |
| 1≤E≤254 | 任意 | **规格化数**：±1.M × 2^(E-偏置) |
| E=255, M=0 | 全0 | **±∞**（无穷大） |
| E=255, M≠0 | 非全0 | **NaN**（非数） |

### 2.5 浮点运算步骤

```
①对阶（小阶向大阶对齐，小阶尾数右移）
②尾数相加减（按定点补码运算）
③规格化（左规或右规）
④舍入（0舍1入 / 恒置1法）
⑤溢出判断（阶码是否超出范围）
```

> 关键：浮点溢出由**阶码**决定；尾数溢出可通过右规修正，阶码溢出才是真正溢出

### 2.6 原码/补码规格化

| 编码 | 规格化判定 |
|------|-----------|
| **原码** | 尾数最高位必须为 **1** |
| **补码** | 尾数最高位与符号位**相反**（正：00.1xxx，负：11.0xxx） |

- **左规**：尾数左移，阶码-1（尾数为 00.0xxx 或 11.1xxx 时）
- **右规**：尾数右移，阶码+1（尾数溢出如 01.xxx 或 10.xxx 时）

## 第三章：程序的转换及机器级表示

### 3.1 程序转换四步

```
源程序(.c) → [预处理] → (.i) → [编译] → 汇编程序(.s) → [汇编] → 目标程序(.o) → [链接] → 可执行文件
```

| 阶段 | 输入 | 输出 | 工具 |
|------|------|------|------|
| **预处理** | .c | .i（展开宏、包含头文件） | cpp |
| **编译** | .i | .s（汇编代码） | cc |
| **汇编** | .s | .o（可重定位目标文件） | as |
| **链接** | .o + 库 | 可执行文件 | ld |

### 3.2 指令格式

一条机器指令 = **操作码(OP)** + **地址码/操作数(OA)**

| 类型 | 格式 | 示例 |
|------|------|------|
| **三地址** | OP DST, SRC1, SRC2 | ADD R1, R2, R3 |
| **二地址** | OP DST, SRC | MOV R1, R2 |
| **一地址** | OP SRC | INC R1 |
| **零地址** | OP | RET |

### 3.3 寻址方式

| 寻址方式 | 操作数获取方式 | 典型应用 |
|----------|---------------|----------|
| **立即寻址** | 操作数直接写在指令中 | 常数赋值 |
| **直接寻址** | 指令给出内存地址 | 全局变量 |
| **间接寻址** | 地址指向的单元存的是操作数地址 | 指针操作 |
| **寄存器寻址** | 操作数在寄存器中 | 局部变量 |
| **寄存器间接寻址** | 寄存器存操作数地址 | 灵活指针 |
| **变址寻址** | EA = 变址寄存器 + 形式地址 | 数组元素 |
| **基址寻址** | EA = 基址寄存器 + 偏移量 | 程序重定位 |
| **相对寻址** | EA = PC + 偏移量 | 跳转/分支指令 |

### 3.4 CISC vs RISC

| 特性 | CISC（x86） | RISC（ARM/MIPS） |
|------|------------|-------------------|
| 指令数量 | 多（200+条） | 少（<100条） |
| 指令长度 | 变长 | 定长（通常4字节） |
| 访存限制 | 不限制 | 仅 Load/Store 指令可访存 |
| 寻址方式 | 多种复杂方式 | 简单寻址 |
| 控制方式 | 微程序控制 | 硬布线控制 |
| 代表 | Intel x86/x64 | ARM、MIPS、RISC-V |

## 第四章：可执行文件的生成与加载执行

### 4.1 ELF 文件格式

Linux/Unix 下可执行文件的标准格式。

**ELF 三种类型**：

| 类型 | 说明 |
|------|------|
| **可重定位文件 (.o)** | 编译后的目标文件，需要链接 |
| **可执行文件** | 链接完成的完整可执行文件 |
| **共享目标文件 (.so)** | 动态链接库 |

### 4.2 符号解析与重定位

| 符号类型 | 说明 |
|----------|------|
| **全局符号** | 本模块定义、可供其他模块引用的符号 |
| **外部符号** | 其他模块定义、本模块引用的符号 |
| **本地符号** | 仅在本模块内部使用的符号（static） |

**重定位**：将可重定位目标文件中的符号地址修正为最终加载后的地址。

### 4.3 程序加载与执行

```
shell → fork() 创建子进程 → execve() 替换为新程序 → _start 入口 → main()
```

### 4.4 CPU 基本组成

```
CPU {
    寄存器组（PC、IR、MAR、MDR、GPRs）
    运算器（ALU）
    控制器（时序逻辑 → 取指/译码/执行控制）
    中断机构
    总线控制逻辑
}
```

**指令周期**：取指 → 译码 → 取操作数 → 执行 → 存结果 → 计算下条地址

## 第五章：程序的存储访问

### 5.1 存储器层次结构

```
寄存器（最快、KB级、CPU内）
  ↕
L1 Cache（SRAM、~1ns、几十KB）
  ↕
L2/L3 Cache（SRAM、~3-10ns、几百KB~几十MB）
  ↕
主存（DRAM、~50-100ns、几GB）
  ↕
外存（SSD/HDD、最慢、TB级）
```

> Cache-主存层次解决**速度**问题（硬件管理），主存-外存层次解决**容量**问题（OS管理）

### 5.2 局部性原理

| 类型 | 原理 | 示例 |
|------|------|------|
| **时间局部性** | 刚访问的数据可能很快再次访问 | 循环中的变量 |
| **空间局部性** | 访问单元附近的单元可能很快被访问 | 数组连续存储 |

### 5.3 Cache（高速缓冲存储器）

#### 地址映射方式

| 方式 | 原理 | 特点 |
|------|------|------|
| **直接映射** | 每个主存块只能映射到唯一 Cache 行 | 简单但冲突率高 |
| **全相联映射** | 主存块可映射到任意 Cache 行 | 命中率最高，实现复杂 |
| **组相联映射** | 每块可放入一组内的 N 个位置 | 折中方案，**最常用** |

#### 替换算法

| 算法 | 说明 |
|------|------|
| **随机 (RAND)** | 随机选择替换，实现简单 |
| **FIFO** | 替换最早调入的块 |
| **LRU** | 替换最久未被访问的块，命中率最高 |
| **LFU** | 替换访问次数最少的块 |

#### 写策略

| 策略 | 原理 | 特点 |
|------|------|------|
| **写回法 (Write Back)** | 只写 Cache，标记脏位，替换时才写回主存 | 速度快，一致性弱 |
| **写贯穿 (Write Through)** | 同时写 Cache 和主存 | 一致性好，速度慢 |

#### 平均访问时间

$$T_{avg} = H \times T_c + (1-H) \times T_m$$

其中 H=命中率，Tc=Cache访问时间，Tm=主存访问时间

### 5.4 虚拟存储器

#### 三种管理方式

| 方式 | 划分方式 | 优点 | 缺点 |
|------|----------|------|------|
| **页式** | 固定大小页面（通常4KB） | 管理简单，无外部碎片 | 有内部碎片 |
| **段式** | 按程序逻辑分段，长度可变 | 符合程序逻辑，便于共享保护 | 有外部碎片 |
| **段页式** | 先分段，段内再分页 | 兼有两者的优点 | 管理复杂 |

#### TLB（快表/转址旁路缓冲）

- 页表的 **Cache**，存放最近使用的页表项
- 采用相联存储器实现
- 加速虚地址 → 物理地址的转换

#### 地址转换流程

```
虚拟地址 = 虚拟页号(VPN) + 页内偏移
        ↓（查TLB → 命中直接用；未命中 → 查页表 → 更新TLB）
物理地址 = 物理页框号(PPN) + 页内偏移
```

## 第六章：程序中 I/O 操作的实现

### 6.1 I/O 系统层次结构

```
用户层 I/O 软件（printf、scanf、read、write）
        ↓
内核空间 I/O 软件（系统调用接口、缓冲区管理）
        ↓
设备驱动程序（与具体硬件交互）
        ↓
中断处理程序（响应硬件中断）
        ↓
I/O 硬件（设备控制器 + 物理设备）
```

### 6.2 I/O 编址方式

| 方式 | 做法 | 特点 |
|------|------|------|
| **统一编址（存储器映射I/O）** | I/O端口与内存统一编址 | 指令丰富，占用地址空间 |
| **独立编址（I/O映射I/O）** | I/O端口独立编址 | 需专用 IN/OUT 指令 |

### 6.3 I/O 控制方式对比

| 方式 | CPU 参与度 | 传输单位 | 效率 | 适用场景 |
|------|-----------|----------|------|----------|
| **程序查询** | 全程轮询 | 逐字节 | 最低 | 简单低速设备 |
| **中断驱动** | 处理中断时介入 | 逐字 | 中等 | 中低速设备 |
| **DMA** | 仅开始/结束介入 | 数据块 | 高 | 高速批量传输（磁盘） |
| **通道** | 几乎不参与 | 一组数据块 | 最高 | 大型机多设备 |

### 6.4 中断处理流程

```
① 中断请求 → ② 中断判优（优先级排队）
→ ③ 关中断 → ④ 保存断点和现场 → ⑤ 开中断
→ ⑥ 执行中断服务程序
→ ⑦ 关中断 → ⑧ 恢复现场 → ⑨ 开中断 → ⑩ 中断返回
```

### 6.5 DMA（直接存储器访问）

**DMA 控制器组件**：

| 寄存器 | 作用 |
|--------|------|
| **DR（数据寄存器）** | 暂存设备与内存间的数据 |
| **MAR（内存地址寄存器）** | 数据在内存中的地址 |
| **DC（数据计数器）** | 剩余要读/写的字节数 |
| **CR（命令/状态寄存器）** | 存放 I/O 命令或设备状态 |

**三种传送方式**：

| 方式 | 做法 | CPU效率 |
|------|------|---------|
| **停止CPU访问** | DMA传输时CPU挂起 | 低 |
| **周期挪用** | DMA"偷"一个存储周期 | 较高（最常用） |
| **CPU与DMA交替** | 分时复用总线 | 最高 |

**DMA vs 中断**：

| 对比 | DMA | 中断 |
|------|-----|------|
| 数据传送 | 外设 ↔ 内存直接 | CPU 搬运 |
| CPU介入 | 仅开始和结束 | 每传输一个数据 |
| 适用场景 | 大批量高速传输 | 少量随机数据 |

### 6.6 磁盘存储

**磁盘访问时间** = 寻道时间 + 旋转延迟时间 + 传输时间

| 指标 | 含义 |
|------|------|
| 记录密度 | 单位长度/面积存储的位数 |
| 存储容量 | 磁盘可存储的总数据量 |
| 平均存取时间 | 寻道 + 旋转延迟 + 传输时间 |
| 数据传输率 | 单位时间读写的数据量 |

**RAID 级别**：

| 级别 | 特点 |
|------|------|
| RAID 0 | 条带化，无冗余，性能最好 |
| RAID 1 | 镜像，100% 冗余 |
| RAID 5 | 分布式奇偶校验，兼顾性能和容错 |
| RAID 10 | RAID 1+0，先镜像再条带 |

## 七、考试重点速记

### 7.1 常考概念

| 考点 | 答案要点 |
|------|----------|
| 冯·诺依曼核心思想 | 二进制、存储程序、五大部件、操作码+地址码 |
| 补码的优点 | 0唯一、减法转加法、符号位参与运算、多表示一个最小负数 |
| IEEE 754 特殊值 | E=M=0→0；E=0,M≠0→非规格化；E=255,M=0→∞；E=255,M≠0→NaN |
| 局部性原理 | 时间局部性（循环）、空间局部性（数组） |
| Cache 三种映射 | 直接映射、全相联映射、组相联映射（最常用） |
| 虚拟存储器三种方式 | 页式（固定大小）、段式（逻辑分段）、段页式（先段后页） |
| 四种 I/O 方式 | 程序查询 → 中断 → DMA → 通道（效率递增） |
| DMA vs 中断 | DMA 用于大批量传输，CPU仅参与开始和结束；中断每字都要 CPU 介入 |

### 7.2 常用公式

| 公式 | 说明 |
|------|------|
| [X]补→[-X]补 | 连同符号位取反+1 |
| IEEE 754 真值 | (-1)^S × 1.M × 2^(E-偏置) |
| Cache 平均访问时间 | H × Tc + (1-H) × Tm |
| 磁盘平均存取时间 | 寻道时间 + 旋转延迟 + 传输时间 |
| 补码范围（n位） | -2^(n-1) ~ 2^(n-1) - 1 |

### 7.3 常见大题题型

- IEEE 754 浮点数表示与转换
- 补码运算与溢出判断
- Cache 映射分析与地址划分
- 页式/段式虚拟地址转换（含 TLB）
